# 소담 알파학원 웹앱

소담 알파학원의 **입시 상담**과 **학생 평가**를 돕는 웹앱입니다. 홈 화면에서 두 도구를 고릅니다.

- **입시 상담 로드맵** — 학생의 학년·월을 입력하면 남은 과학·수학 과정과 월별 시간표를 만들어 학부모님께 전달(인쇄/PDF/이미지)
- **학생 개별 평가** — 진단·단원 평가를 채점하고 유형별 정답률 리포트로 취약점 파악

> 데모: `https://devrobertson.github.io/alpha-sodam-academy-schedule/`

브라우저에 자동 저장(localStorage)되며, 데이터는 JSON으로 백업·복원할 수 있습니다. **구글 로그인 시 Firebase(Firestore)로 실시간 동기화**되어 여러 기기·선생님이 같은 데이터를 공유합니다(오프라인 지원, 로그인 안 하면 로컬 저장만).

## 1) 입시 상담 로드맵

상담 선생님이 **학생 이름 · 현재 학년 · 상담 월**을 입력하면 자동으로 만들어집니다. (**학교·연락처**는 선택 입력이며, **동명이인 구분**에 씁니다.)

1. **남은 과목 로드맵** — 상담 월 기준으로 아직 안 들은 과정만 시험 전까지 월별로 배치
2. **월별 시간표** — 매달의 주간 시간표(담당 선생님 포함)
3. **내보내기** — 인쇄/PDF, 또는 페이지별 PNG를 ZIP으로 저장
4. **학생 시간표 저장 / 불러오기** — 맨 위 **학생** 드롭다운으로 저장된 학생을 불러오거나 새로 만들고, **💾 학생 시간표 저장**을 누르면 그 학생의 로드맵·시간표(포함 과목·시작월 이동·세션 요일/시간)가 저장되고 **학생 개별 평가 탭에 자동 등록**됩니다. 같은 이름의 수업도 **학생마다 다른 요일·시간**으로 저장됩니다.

### 상호작용

- **로드맵에서 과목 막대 클릭** → 선택, **좌우 드래그** → 수강 시작 월 이동(월별 시간표에 자동 반영), 선택 후 **✕** → 로드맵에서 제거
- **월별 시간표에서 `◀ ▶`로 달 이동**, 수업 블록을 드래그해 요일·시간 조정(시간 충돌 시 경고)
- 한 과정이 **주 2회 이상**이면 세션마다 블록이 표시되고 세션별로 따로 옮길 수 있음

### 내보내기(인쇄 / 이미지)

- **인쇄 / PDF** — 1페이지는 "로드맵"(로드맵만), 2페이지부터 가로 방향으로 **한 장에 월 시간표 2개(좌·우)**를 마지막 수업 달까지. 각 시간표는 상하 중앙 정렬, 기본 13–19시(수업에 맞춰 확장).
- **이미지(PNG·ZIP)** — 위 인쇄 페이지를 **페이지별 PNG**로 만들어 ZIP 한 개로 저장.

### 관리(Admin) 탭

과정 데이터를 코드 수정 없이 직접 편집합니다.

- 과정의 **개설 월·기간 · 유형 · 담당 선생님** 편집, 과정 추가/삭제
- **0.5개월(반개월) 단위** 관리 — 시작·종료의 **½** 체크로 그 달 중순부터 시작/중순에 종료(로드맵 막대가 반 칸으로 렌더). 상담 화면 드래그도 0.5개월 스냅. (내부적으로 `spanStart`/`spanMonths` 사용)
- 과정마다 **수업 세션(주 N회)** 추가/삭제 및 세션별 요일·시작·종료 편집
- **로드맵 포함 여부** 체크박스, 트랙 필터, 전체 선택/해제
- **JSON 내보내기/가져오기**로 백업·복원·다른 기기 이전, **기본값 복원**
- 변경 내용은 브라우저에 자동 저장(`asg.store.v4`)

### 도메인 개념

- **월(月) 단위 타임라인**: 초4 3월 ~ 중3 2월 = 72칸
- **과목**: 과학·수학 (**과학=블루/네이비, 수학=그린** 계열로 과목 구분, 교과 유형 **초등교과 / 중등교과 / 고등교과**로 명암 구분)
- **과정(course)**: 학원 수업. 개설 월·기간, 수업 세션(요일·시작·종료, 주 N회), 담당 선생님
- **상태**: 상담 월 기준으로 `완료 / 진행중 / 예정`으로 계산되며, 로드맵에는 **완료를 제외한 남은 과목**만 나옴

기본 과학 커리큘럼(11개 과정):

- **중등**: 과학 중1-1 · 중1-2 · 중2-1 · 중2-2 · 중3-1 · 중3-2
- **고등**: 고등화학 · 화학문제풀이 · 고등물리학 · 고등 통합과학 1 · 고등 통합과학 2

기본 수학 커리큘럼(18개 과정) — **선행 순차 배치**(초4 3월부터 순서대로 이어짐):

- **학기별**: 수학 초4-1 ~ 중3-2 (초4·초5·초6·중1·중2·중3, 각 1·2학기 = 12개)
- **고등(교과목)**: 공통수학1 · 공통수학2 · 대수 · 미적분 · 기하 · 확률과통계
- **과정별 기간**: 초등 2.5개월 · 중등 3.5개월 · 고등 5개월. 반개월까지 **정밀 배치**(과정에 `spanStart`/`spanMonths` 부여)되어 로드맵 막대가 정확히 2.5·3.5·5칸으로 그려집니다.
- 이 과정들은 **학생 개별 평가**의 수강 수업·시험지 수업 목록에도 그대로 나타납니다.

## 2) 학생 개별 평가

진단평가부터 단원평가까지 학생별 채점 자료를 쌓고, 유형별 리포트를 봅니다. (자동 저장 `asg.assess.v1`, JSON 백업)

**수업(class)** 개념으로 묶입니다. 수강 수업 = 입시 상담 로드맵의 과정. **진단테스트**는 모든 학생 공통이라 수강 목록에는 없고, 채점 입력에서만 수업으로 제공됩니다.

- **학생 관리** — 학생은 **입시 상담 로드맵에서 학생 시간표를 저장하면 자동 등록**됩니다(여기서 직접 추가는 하지 않음). 등록된 학생의 이름·학년·**학교·연락처**(동명이인 구분용, 선택)·**듣는 수업**·메모를 확인·수정·삭제하고, **학생 명단 CSV 내려받기**로 백업합니다.
- **시험지 관리** — **CSV 업로드**로 시험지 정의(CSV가 곧 "문항번호 → 유형" 표). **여러 CSV 동시 업로드**(미리보기 후 모두 저장) 및 **체크박스 다중 삭제** 지원. 업로드 시 어느 **수업**의 시험지인지 지정(CSV의 수업 열 자동 인식, 없으면 진단평가는 진단테스트가 기본).
- **채점 입력** — **학생 → 수업 → 시험지** 순으로 선택. 학생을 고르면 그 학생이 듣는 수업(+진단테스트)이 나오고, 수업을 고르면 그 수업의 시험지만 표시. 문항별 **O/X 그리드**(전체 O/X·초기화, 실시간 정답률), 저장된 채점 재수정. **채점 CSV 내려받기/올리기**(학생·수업·시험지·응시일·문항번호·OX).
- **리포트** — **유형별 정답률** 막대 + **레이더 차트** + 응시 이력. **리포트에 포함할 시험을 직접 선택**하거나 **기간(시작~종료)으로 선택**하면 그 기간에 응시한 시험이 모두 포함됩니다. **✍ 선생님 종합 의견·추천 수업**을 직접 적으면 리포트와 함께 **PDF로 출력**(학부모 전달용)됩니다(저장하지 않는 임시 입력 — 학생을 바꾸면 비워집니다). PDF는 **레터헤드(로고·학원명)·학생 정보 표·발행 푸터·직인**을 갖춘 공식 리포트 형식으로 나오며, **뒤에 첨부 페이지**(`app/src/assets/report-append.pdf`)가 자동으로 붙습니다(파일을 교체하면 그 PDF가, 없으면 리포트만 저장). 병합은 `pdf-lib` 사용.

### 시험지 CSV 형식

문항번호·유형이 **필수**, 나머지(시험지·과목·정답·배점·**수업**)는 선택입니다. 헤더 이름으로 매핑하므로 열 순서는 무관하고 한글/영문 별칭을 허용합니다. 배점은 소수점 첫째 자리까지 가능합니다. **수업** 열에 수업 이름을 넣으면 업로드 시 그 수업으로 자동 지정됩니다(없으면 진단평가는 진단테스트가 기본).

```csv
시험지,수업,과목,문항번호,유형,정답,배점
중2 1차 진단평가,진단테스트,과학,1,자료 해석,3,3.5
중2 1차 진단평가,진단테스트,과학,2,분석·추론,1,3
```

> 과학 문항의 **유형**은 학생의 **사고 역량** 축(핵심 6종: 개념 이해 · 자료 해석 · 적용·계산 · 분석·추론 · 탐구·실험 설계 · 비교·분류)을 사용합니다. 자세한 설명·보완 훈련은 앱의 **사용 설명서**에 있습니다.

### 학생 명단 CSV / 채점 CSV

```csv
이름,학년,듣는수업
이영희,중2,진단테스트;고등화학
```

```csv
학생,수업,시험지,응시일,문항번호,OX
이영희,진단테스트,중2 1차 진단평가,2026-07-01,1,O
이영희,진단테스트,중2 1차 진단평가,2026-07-01,2,X
```

## 로고 · 이미지 업로드

- `app/src/assets/brand/`에 `logo.png`(또는 `.jpg/.svg/.webp`)를 올리면 **홈·헤더 로고로 자동 적용**됩니다. 파일이 없으면 기본 알파(α) 로고가 표시됩니다.
- 같은 폴더에 `seal.png`(또는 `.jpg/.svg/.webp`)를 올리면 **리포트 PDF 하단 직인**으로 쓰입니다. 파일이 없으면 **자동 생성 도장**(빨간 원형)이 표시됩니다.
- `app/src/assets/ui/`에는 UI에 넣을 그림이나 참고 이미지를 올릴 수 있습니다.
- 각 폴더의 `README.md`에 사용법이 있습니다. 배포 반영에는 재빌드가 필요합니다.

## 기술 스택

- Vite + React + TypeScript
- 라우팅: 해시 기반(`#/consult`, `#/assessment`) — GitHub Pages 새로고침 대응
- 드래그앤드롭: `@dnd-kit/core`(시간표), 포인터 이벤트(로드맵 월 이동)
- 차트/로드맵: 외부 라이브러리 없이 **SVG 직접 렌더**
- 내보내기: `html-to-image`(PNG/JPEG) + `jszip`(ZIP) + `jspdf`(리포트 PDF, 동적 로드)
- 클라우드 동기화: `firebase`(구글 로그인 + Firestore 실시간 동기화, 오프라인 캐시). 설정은 `src/lib/firebase.ts`, 접근 통제는 Firestore 보안 규칙(허용 이메일)
- 테스트: Vitest

## 개발

소스는 모두 `app/` 폴더에 있습니다. 명령은 `app/`에서 실행하세요.

```bash
cd app
npm install
npm run dev      # 개발 서버
npm run test     # 유닛 테스트(Vitest)
npm run build    # 프로덕션 빌드 → 레포 루트(index.html, assets/)로 출력
npm run preview  # 빌드 결과 미리보기
```

## 배포 (GitHub Pages · 워크플로우 없음)

빌드 결과물을 **레포 루트에 커밋**하는 방식입니다. GitHub Pages가 `main` 루트를 그대로 서빙합니다. (Actions 워크플로우·Pages 설정 변경 불필요)

코드를 수정한 뒤 배포하려면:

```bash
cd app && npm run build           # 루트의 index.html / assets/ 갱신
cd .. && git add -A && git commit -m "build" && git push
```

> Vite/React 앱은 **빌드된 결과물만** 브라우저에서 동작합니다. 소스만 push하면 흰 화면이 되니, 반드시 위처럼 빌드 후 결과물을 커밋하세요.
>
> 레포명이 바뀌면 `app/vite.config.ts`의 `base`를 `/<레포명>/`로 맞춘 뒤 다시 빌드·커밋하세요. (현재 `/alpha-sodam-academy-schedule/`)
