# 알파학원 입시 상담 로드맵

입시 **상담용 웹앱**입니다. 상담 온 학생의 정보를 입력하면, **목표 학교 합격까지 남은 과목**과 **월별 시간표**를 만들어 학부모님께 바로 전달(인쇄/PDF/이미지)할 수 있습니다.

> 데모: `https://devrobertson.github.io/alpha-sodam-academy-schedule/`

---

## 무엇을 하나요

상담 선생님이 다음만 입력합니다.

- 학생 이름(선택), **현재 학년**, **상담 월**
- **수학 진도**(완료한 단계), **과학 진도**(중등 / 고등 진입)
- **목표 학교** 1곳 (영재학교 · 과학고 · 국제고 · 외고 · 전사고)

그러면 자동으로:

1. **남은 과목 로드맵** — 목표 학교 합격까지 아직 안 들은 과정만 시험 전까지 월별로 배치
2. **월별 시간표** — 매달의 주간 시간표(담당 선생님 포함)
3. **내보내기** — 학생 정보·진도·상담일이 들어간 최종본을 인쇄/PDF/PNG로 저장

### 상호작용

- **로드맵에서 과목 막대를 좌우로 드래그** → 수강 시작 월 이동 → 월별 시간표에 자동 반영
- **월별 시간표에서 `◀ ▶`로 달 이동**, 수업 블록을 드래그해 요일·시간 조정 (시간 충돌 시 경고)
- 한 과정이 **주 2회 이상**이면 세션마다 블록이 표시되고 세션별로 따로 옮길 수 있음

## 도메인 개념

- **월(月) 단위 타임라인**: 초5 3월 ~ 중3 2월 = 60칸. (계절 아님)
- **목표 학교(track)**: 영재학교 · 과학고 · 국제고 · 외고 · 전사고. 로드맵은 선택한 1곳 기준.
- **과정(course)**: 학원 수업. 개설 월·기간, 수업 세션(요일·시작·종료, 주 N회), 담당 선생님을 가짐.
  - `KMO`는 **KMO 대수 / 기하 / 정수 / 조합** 4과정으로 분리되어 있습니다.
  - **교과(수학 교과 · 과학 교과)** 도 `트랙 = 공통` 과정으로 관리되어 모든 목표 학교 학생에게 공통 표시됩니다.
- **유형(8종)**: 영재학교입시 · 과학고입시 · 외고입시 · 국제고입시 · 고등내신 · 과학고내신 · 중등선행 · 고등선행
- **상태**: 상담 월 기준으로 과정이 `완료 / 진행중 / 예정`으로 계산되며, 로드맵에는 **완료를 제외한 남은 과목**만 나옵니다.

## 관리(Admin) 탭

과정 데이터를 직접 편집합니다. 코드를 고칠 필요가 없습니다.

- 과정의 **개설 월·기간 · 유형 · 과목 · 담당 선생님** 편집, 과정 추가/삭제
- 과정마다 **수업 세션(주 N회)** 추가/삭제 및 세션별 요일·시작·종료 편집
- 교과 진도 **투영 속도**(개월/단원) 설정
- 변경 내용은 **브라우저(localStorage)에 자동 저장**
- **JSON 내보내기/가져오기** 로 백업·복원·다른 기기 이전, **기본값 복원**

## 기술 스택

- Vite + React + TypeScript
- 드래그앤드롭: `@dnd-kit/core` (시간표), 포인터 이벤트(로드맵 월 이동)
- 차트: 외부 라이브러리 없이 **SVG 직접 렌더**
- 이미지 내보내기: `html-to-image`
- 테스트: Vitest

## 개발

소스는 모두 `app/` 폴더에 있습니다. 명령은 `app/`에서 실행하세요.

```bash
cd app
npm install
npm run dev      # 개발 서버
npm run test     # 로직 테스트
npm run build    # 프로덕션 빌드 → 레포 루트(index.html, assets/)로 출력
npm run preview  # 빌드 결과 미리보기
```

## 배포 (GitHub Pages · 워크플로우 없음)

빌드 결과물을 **레포 루트에 커밋**하는 방식입니다. GitHub Pages가 `main` 루트를 그대로 서빙합니다. (Actions 워크플로우·Pages 설정 변경 불필요)

코드를 수정한 뒤 배포하려면:

```bash
cd app && npm run build           # 루트의 index.html / assets/ 갱신
cd .. && git add -A && git commit -m "build" && git push
```

> Vite/React 앱은 **빌드된 결과물만** 브라우저에서 동작합니다. 소스만 push하면 흰 화면이 되니, 반드시 위처럼 빌드 후 결과물을 커밋하세요.
>
> 레포명이 바뀌면 `app/vite.config.ts`의 `base`를 `/<레포명>/`로 맞춘 뒤 다시 빌드·커밋하세요. (현재 `/alpha-sodam-academy-schedule/`)
