# 소담 알파학원 웹앱

소담 알파학원의 **입시 상담**과 **학생 평가**를 돕는 웹앱입니다. 홈 화면에서 두 도구를 고릅니다.

- **입시 상담 로드맵** — 학생의 학년·월을 입력하면 남은 과학 과정과 월별 시간표를 만들어 학부모님께 전달(인쇄/PDF/이미지)
- **학생 개별 평가** — 진단·주별 평가를 채점하고 유형별 정답률 리포트로 취약점 파악

> 데모: `https://devrobertson.github.io/alpha-sodam-academy-schedule/`

브라우저에 자동 저장(localStorage)되며, 데이터는 JSON으로 백업·복원할 수 있습니다. 별도 서버·로그인은 없습니다.

---

## 1) 입시 상담 로드맵

상담 선생님이 **학생 이름(선택) · 현재 학년 · 상담 월**만 입력하면 자동으로 만들어집니다.

1. **남은 과목 로드맵** — 상담 월 기준으로 아직 안 들은 과학 과정만 시험 전까지 월별로 배치
2. **월별 시간표** — 매달의 주간 시간표(담당 선생님 포함)
3. **내보내기** — 인쇄/PDF, 또는 페이지별 PNG를 ZIP으로 저장

### 상호작용

- **로드맵에서 과목 막대 클릭** → 선택, **좌우 드래그** → 수강 시작 월 이동(월별 시간표에 자동 반영), 선택 후 **✕** → 로드맵에서 제거
- **월별 시간표에서 `◀ ▶`로 달 이동**, 수업 블록을 드래그해 요일·시간 조정(시간 충돌 시 경고)
- 한 과정이 **주 2회 이상**이면 세션마다 블록이 표시되고 세션별로 따로 옮길 수 있음

### 내보내기(인쇄 / 이미지)

- **인쇄 / PDF** — 1페이지는 "로드맵"(로드맵만), 2페이지부터 가로 방향으로 **한 장에 월 시간표 2개(좌·우)**를 마지막 수업 달까지. 각 시간표는 상하 중앙 정렬, 기본 13–19시(수업에 맞춰 확장).
- **이미지(PNG·ZIP)** — 위 인쇄 페이지를 **페이지별 PNG**로 만들어 ZIP 한 개로 저장.

### 관리(Admin) 탭

과정 데이터를 코드 수정 없이 직접 편집합니다.

- 과정의 **개설 월·기간 · 유형 · 담당 선생님** 편집, 과정 추가/삭제
- 과정마다 **수업 세션(주 N회)** 추가/삭제 및 세션별 요일·시작·종료 편집
- **로드맵 포함 여부** 체크박스, 트랙 필터, 전체 선택/해제
- **JSON 내보내기/가져오기**로 백업·복원·다른 기기 이전, **기본값 복원**
- 변경 내용은 브라우저에 자동 저장(`asg.store.v4`)

### 도메인 개념

- **월(月) 단위 타임라인**: 초5 3월 ~ 중3 2월 = 60칸
- **과목**: 과학 (교과 유형 **중등교과 / 고등교과**로 색 구분 — 중등=블루, 고등=네이비)
- **과정(course)**: 학원 수업. 개설 월·기간, 수업 세션(요일·시작·종료, 주 N회), 담당 선생님
- **상태**: 상담 월 기준으로 `완료 / 진행중 / 예정`으로 계산되며, 로드맵에는 **완료를 제외한 남은 과목**만 나옴

기본 과학 커리큘럼(11개 과정):

- **중등**: 과학 중1-1 · 중1-2 · 중2-1 · 중2-2 · 중3-1 · 중3-2
- **고등**: 고등화학 · 화학문제풀이 · 고등물리학 · 고등 통합과학 1 · 고등 통합과학 2

## 2) 학생 개별 평가

진단평가부터 주별평가까지 학생별 채점 자료를 쌓고, 유형별 리포트를 봅니다. (자동 저장 `asg.assess.v1`, JSON 백업)

**수업(class)** 개념으로 묶입니다. 수업 목록 = 입시 상담 로드맵의 과정 + **진단테스트**.

- **학생 관리** — 학생 추가/수정/삭제·메모, **듣는 수업**(여러 개) 선택. **학생 명단 CSV 내려받기/올리기**(이름·학년·듣는수업).
- **시험지 관리** — **CSV 업로드**로 시험지 정의(CSV가 곧 "문항번호 → 유형" 표). 업로드 시 어느 **수업**의 시험지인지 지정(진단평가는 진단테스트가 기본).
- **채점 입력** — **수업 → 학생 → 시험지** 순으로 선택하면 그 수업의 학생·시험지만 표시. 문항별 **O/X 그리드**(전체 O/X·초기화, 실시간 정답률), 저장된 채점 재수정. **채점 CSV 내려받기/올리기**(학생·수업·시험지·응시일·문항번호·OX).
- **리포트** — **유형별 정답률** 막대 + **레이더 차트** + 응시 이력. **리포트에 포함할 시험을 선택**할 수 있고, **PDF로 저장**(학부모 전달용).

### 시험지 CSV 형식

문항번호·유형이 **필수**, 나머지는 선택입니다. 헤더 이름으로 매핑하므로 열 순서는 무관하고 한글/영문 별칭을 허용합니다. 배점은 소수점 첫째 자리까지 가능합니다.

```csv
시험지,과목,문항번호,유형,정답,배점
중2 1차 진단평가,과학,1,그래프/자료 해석형,3,3.5
중2 1차 진단평가,과학,2,새 개념 정의형,1,3
```

### 학생 명단 CSV / 채점 CSV

```csv
이름,학년,듣는수업
이영희,중2,진단테스트;고등화학
```

```csv
학생,수업,시험지,응시일,문항번호,OX
이영희,진단테스트,중2 1차 진단평가,2026-07-01,1,O
이영희,진단테스트,중2 1차 진단평가,2026-07-01,2,X
```

## 기술 스택

- Vite + React + TypeScript
- 라우팅: 해시 기반(`#/consult`, `#/assessment`) — GitHub Pages 새로고침 대응
- 드래그앤드롭: `@dnd-kit/core`(시간표), 포인터 이벤트(로드맵 월 이동)
- 차트/로드맵: 외부 라이브러리 없이 **SVG 직접 렌더**
- 내보내기: `html-to-image`(PNG/JPEG) + `jszip`(ZIP) + `jspdf`(리포트 PDF, 동적 로드)
- 테스트: Vitest

## 개발

소스는 모두 `app/` 폴더에 있습니다. 명령은 `app/`에서 실행하세요.

```bash
cd app
npm install
npm run dev      # 개발 서버
npm run test     # 유닛 테스트(Vitest)
npm run build    # 프로덕션 빌드 → 레포 루트(index.html, assets/)로 출력
npm run preview  # 빌드 결과 미리보기
```

## 배포 (GitHub Pages · 워크플로우 없음)

빌드 결과물을 **레포 루트에 커밋**하는 방식입니다. GitHub Pages가 `main` 루트를 그대로 서빙합니다. (Actions 워크플로우·Pages 설정 변경 불필요)

코드를 수정한 뒤 배포하려면:

```bash
cd app && npm run build           # 루트의 index.html / assets/ 갱신
cd .. && git add -A && git commit -m "build" && git push
```

> Vite/React 앱은 **빌드된 결과물만** 브라우저에서 동작합니다. 소스만 push하면 흰 화면이 되니, 반드시 위처럼 빌드 후 결과물을 커밋하세요.
>
> 레포명이 바뀌면 `app/vite.config.ts`의 `base`를 `/<레포명>/`로 맞춘 뒤 다시 빌드·커밋하세요. (현재 `/alpha-sodam-academy-schedule/`)
